​ 最近接触到了一个很不错的接口文档解决方案，即EasyJavaDoc + smart-doc + torna

通过EasyJavaDoc + smart-doc可以很方便的无侵入的给接口，类等添加注释。相较于swagger，不必写繁琐的注解，直接通过快捷键，能够生成相关注释。

结合torna，能够极其方便的将文档发布在文档后台系统中，并支持权限管理，很适合企业级应用。

本篇博文便介绍smart-doc + torna的配置与部署。

EasyJavaDoc官方文档：https://github.com/starcwang/easy_javadoc

smart-doc官方文档：https://smart-doc-group.github.io/#/zh-cn/start/quickstart

torna官方文档：http://www.torna.cn/

基本环境

  • Java8
  • Maven3
  • node.js16
  • mysql5.7

使用

torna的使用需要结合smart-doc进行文档推送，在详细讲解smart-doc的安装及配置之后再予以实操，这里先贴一个很重要的配置，为smart-doc的推送做准备

EasyJavaDoc

EasyJavaDoc在这个搭配里扮演的角色是一键生成注释，省的我们一点点的写注释，减轻写注释的负担。

效果

一键生成文档

效果如图，通过一个组合键：ctrl + \即可生成以上文档，该文档完美符合smart-doc接口标准，因此选择EasyJavaDoc+smart-doc组合生成文档，真觉得这两天生是一对

安装步骤

EasyJavaDoc是一个Idea插件，因此可以直接在Idea插件市场进行搜索

使用

在接口、类、属性上使用ctrl + \一键生成文档

自定义配置

在方法注释模板中，默认并没有增加上@author注解，生成的smart-doc接口上也不会有作者信息

因此选择自定义模板，修改位置在Idea -> Settings ->Other Settings，直接贴图吧

其他配置大家可以自己研究，展示一下我自定义的方法注释模板

其中，作者等自定义变量需要手动添加，提供以下内置变量

配置详情如下

/**
 *$DOC$
 *
 *$PARAMS$
 *$RETURN$
 *$THROWS$
 *@author $AUTHOR$
 */

Smart-doc

smart-doc是重头戏了，我们接口文档页面上所有能看到的配置项如维护者、接口请求参数、响应参数等等，都是smart-doc帮我们解析然后生成页面的。

因此smart-doc有相当之多的约定配置需要我们遵守，我们添加的标签与注释决定了文档页面上展示的内容与效果

效果

smart-doc支持生成html、markdown、web等等形式的文档，这里贴一个html的吧

安装步骤

具体配置还是参考官方文档， 可以基于maven或者gradle插件进行安装，这里只讲解自己的安装步骤

samrt-doc官方文档

  1. 添加maven插件

    <plugin>
    <groupId>com.github.shalousun</groupId>
    <artifactId>smart-doc-maven-plugin</artifactId>
    <version>[最新版本]</version>
    <configuration>
        <!--指定生成文档的使用的配置文件,配置文件放在自己的项目中-->
        <configFile>./src/main/resources/smart-doc.json</configFile>
        <!--指定项目名称-->
        <projectName>测试</projectName>
        <!--smart-doc实现自动分析依赖树加载第三方依赖的源码，如果一些框架依赖库加载不到导致报错，这时请使用excludes排除掉-->
        <excludes>
            <!--格式为：groupId:artifactId;参考如下-->
            <!--也可以支持正则式如：com.alibaba:.* -->
            <exclude>com.alibaba:fastjson</exclude>
        </excludes>
        <!--includes配置用于配置加载外部依赖源码,配置后插件会按照配置项加载外部源代码而不是自动加载所有，因此使用时需要注意-->
        <!--smart-doc能自动分析依赖树加载所有依赖源码，原则上会影响文档构建效率，因此你可以使用includes来让插件加载你配置的组件-->
        <includes>
            <!--格式为：groupId:artifactId;参考如下-->
            <!--也可以支持正则式如：com.alibaba:.* -->
            <include>com.alibaba:fastjson</include>
        </includes>
    </configuration>
    <executions>
        <execution>
            <!--如果不需要在执行编译时启动smart-doc，则将phase注释掉-->
            <phase>compile</phase>
            <goals>
                <!--smart-doc提供了html、openapi、markdown等goal，可按需配置-->
                <goal>html</goal>
            </goals>
        </execution>
    </executions>
</plugin>

    这里有个坑，如果自己的工程依赖到了其他工程的类，如入参、响应等，这里典型应用有dubbo接口，就需要去掉上文的<excludes>及<includes>标签，具体配置如下：

    <plugin>
    <groupId>com.github.shalousun</groupId>
    <artifactId>smart-doc-maven-plugin</artifactId>
    <version>2.2.7</version>
    <configuration>
        <!--指定生成文档的使用的配置文件,配置文件放在自己的项目中-->
        <configFile>./src/main/resources/smart-doc.json</configFile>
        <!--指定项目名称-->
        <projectName>测试</projectName>

    </configuration>
    <executions>
        <execution>
            <!--如果不需要在执行编译时启动smart-doc，则将phase注释掉-->
            <phase>compile</phase>
            <goals>
                <!--smart-doc提供了html、openapi、markdown等goal，可按需配置-->
                <goal>html</goal>
            </goals>
        </execution>
    </executions>
</plugin>

  "coverOld": true,  //是否覆盖旧的文件，主要用于mardown文件覆盖
  "createDebugPage": true,//@since 2.0.0 smart-doc支持创建可以测试的html页面，仅在AllInOne模式中起作用。
  "packageFilters": "",//controller包过滤，多个包用英文逗号隔开，2.2.2开始需要采用正则：com.test.controller.*
  "md5EncryptedHtmlName": false,//只有每个controller生成一个html文件是才使用
  "style":"xt256", //基于highlight.js的代码高设置,可选值很多可查看码云wiki，喜欢配色统一简洁的同学可以不设置
  "projectName": "smart-doc",//配置自己的项目名称，不设置则插件自动获取pom中的projectName
  "framework": "spring",//smart-doc默认支持spring和dubbo框架的文档，使用默认框架不用配置，当前支持spring、dubbo、JAX-RS、solon 
  "skipTransientField": true,//目前未实现
  "sortByTitle":false,//接口标题排序，默认为false,@since 1.8.7版本开始

  "displayActualType":false,//配置true会在注释栏自动显示泛型的真实类型短类名，@since 1.9.6
  "appKey": "20201216788835306945118208",// torna平台对接appKey,, @since 2.0.9
  "appToken": "c16931fa6590483fb7a4e85340fcbfef", //torna平台appToken,@since 2.0.9
  "secret": "W.ZyGMOB9Q0UqujVxnfi@.I#V&tUUYZR",//torna平台secret，@since 2.0.9
  "openUrl": "http://localhost:7700/api",//torna平台地址，填写自己的私有化部署地址@since 2.0.9
  "debugEnvName":"测试环境", //torna环境名称
  "debugEnvUrl":"http://127.0.0.1",//推送torna配置接口服务地址
  "tornaDebug":false,//启用会推送日志
  "ignoreRequestParams":[ //忽略请求参数对象，把不想生成文档的参数对象屏蔽掉，@since 1.9.2
     "org.springframework.ui.ModelMap"
   ],
  "dataDictionaries": [{ //配置数据字典，没有需求可以不设置
      "title": "http状态码字典", //数据字典的名称
      "enumClassName": "com.power.common.enums.HttpCodeEnum", //数据字典枚举类名称

标签

文档的内容解析便是基于标签，这里就不在讲解了，仅列举几个，可以参考官方文档

smart-doc标签

  • @ignore
  • @required
  • @mock
  • @download

使用规范

  • 强制：实体类(特别是入参和响应)字段一定要有注释，如果是入参建议加上@mock，便于前端联调
  • 强制：controller及每个接口必须添加描述信息
  • 建议：controlelr每个接口加上@author，方便前端找人联调

以上规范均为个人使用过程中习惯与思考，因人而异

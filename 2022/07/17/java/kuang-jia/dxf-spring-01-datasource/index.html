Actuator

什么是actuator

actuator给我们提供了一系列endpoint（端点）允许我们在启动springboot web项目后，能够看到如url资源、beans、健康状态等等，如下图所示：

通过访问路径：http://127.0.0.1:8080/actuator能够查看默认情况下能够访问的资源

如何使用

通过引入pom文件

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

如何开启更多的actuator

默认情况下，只开启了以上三个endpoint，（每个springboot版本不一致），如果需要开启更多的endpoint，可以通过以下配置进行按需开启

# 可以這樣寫，就會開啟所有endpoints(不包含shutdown)
management.endpoints.web.exposure.include=*

# 也可以這樣寫，就只會開啟指定的endpoint，因此此處只會再額外開啟/actuator/beans和/actuator/mappings
management.endpoints.web.exposure.include=beans,mappings

# exclude可以用來關閉某些endpoints
# exclude通常會跟include一起用，就是先include了全部，然後再exclude /actuator/beans這個endpoint
management.endpoints.web.exposure.exclude=beans
management.endpoints.web.exposure.include=*

# 如果要開啟/actuator/shutdown，要額外再加這一行
management.endpoint.shutdown.enabled=true

使用建议

由于endpoint给予了用户查看web服务核心信息的能力，因此不建议开启太多的endpoint，开启时也要充分考虑到安全问题，特别的shutdown endpont允许了用户通过该endpoint关闭应用程序，这个是相当危险的

H2数据库

H2是一个内存级别的数据库，采用纯JAVA语言编写，在Spring Boot中，可以很方便对其进行集成

<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>

获取连接

在不用做任何配置（仅仅引入依赖）的情况下，我们可以自动连接该数据库（基于内存的，spring boot项目一启动该数据库便会运行）

可以打印其连接信息：

HikariProxyConnection@1729904998 wrapping conn0: url=jdbc:h2:mem:879a3d17-80c8-4733-a7c7-5b50f74755e9 user=SA

spring boot 2以上默认是hikari连接池，打印的是H2数据库的0号连接，地址为url内容，用户名为SA

定义schema

由于H2数据库是内存数据库，因此我们需要定义好表结构，应用启动后，直接读取schema配置并创建相关表

配置路径位于resoucre文件夹下的schema.sql文件（需要我们创建）

create table foo
(
    id  int,
    bar varchar(255)
);

定义data

创建完表之后，可能需要对数据进行初始化，我们可以通过定义data.sql文件，对刚才创建的表插入部分数据

insert into foo
values (1, '张三'),
       (2, '李四');

多数据源配置

开发场景中，我们可能有多个数据源，数据源的配置本质上也就是bean的配置，可以具体为DataSource、JDBCTemplate、TransactionManage的配置；其中JDBCTemplate、TransactionManage都是基于DataSource而确定的，本质上还是配置多个的DataSource

配置多数据源

  1. 在配置文件中，声明多个DataSource

    # 数据源1 foo
foo.datasource.url=jdbc:h2:mem:foo
foo.datasource.username=sa
foo.datasource.password=

# 数据源2 bar
bar.datasource.url=jdbc:h2:mem:bar
bar.datasource.username=sa
bar.datasource.password=

  2. 编写配置类，声明多个DataSource以及对应的TransactionManage等

    @Configuration
@Slf4j
public class MutiDataSourceConfig {
    @Bean
    @ConfigurationProperties("foo.datasource")
    public DataSourceProperties fooDataSourceProperties() {
        return new DataSourceProperties();
    }

    /**
     * foo数据源
     *
     * @return {@link DataSource }
     * @author zhuzy
     * @date 2022/07/17 20:01
     */
    @Bean
    public DataSource fooDataSource() {
        DataSourceProperties dataSourceProperties = fooDataSourceProperties();
        log.info("foo dataSourceProperties: {}", dataSourceProperties.getUrl());
        return dataSourceProperties.initializeDataSourceBuilder().build();
    }

    /**
     * foo 事务管理器
     *
     * @param fooDataSource foo数据源
     * @return {@link PlatformTransactionManager }
     * @author zhuzy
     * @date 2022/07/17 20:03
     */
    @Bean
    @Resource
    public PlatformTransactionManager fooTxManager(DataSource fooDataSource) {
        return new DataSourceTransactionManager(fooDataSource);
    }
    
    @Bean
    public JdbcTemplate fooJdbcTemplate() {
        return new JdbcTemplate(fooDataSource());
    }
    
    



    @Bean
    @ConfigurationProperties("bar.datasource")
    public DataSourceProperties barDataSourceProperties() {
        return new DataSourceProperties();
    }

    /**
     * bar数据源
     *
     * @return {@link DataSource }
     * @author zhuzy
     * @date 2022/07/17 20:02
     */
    @Bean
    public DataSource barDataSource() {
        DataSourceProperties dataSourceProperties = barDataSourceProperties();
        log.info("bar dataSourceProperties: {}", dataSourceProperties.getUrl());
        return dataSourceProperties.initializeDataSourceBuilder().build();
    }

    /**
     * bar 事务管理器
     *
     * @param barDataSource bar数据源
     * @return {@link PlatformTransactionManager }
     * @author zhuzy
     * @date 2022/07/17 20:03
     */
    @Bean
    @Resource
    public PlatformTransactionManager barTxManager(DataSource barDataSource) {
        return new DataSourceTransactionManager(barDataSource);
    }
    
    @Bean
    public JdbcTemplate barJdbcTemplate() {
        return new JdbcTemplate(barDataSource());
    }
}

    ​ 这里我声明了两个数据源，并针对foo以及bar数据源，分别声明了他们的事务管理器以及JdbcTemplate

使用多数据源

多数据源有两种方式可以进行使用

  • 方法1：声明其中一组数据源（DataSource、TransactionManager、JdbcTemplate）作为Primary的bean，让Spring Boot在注入Bean的时候，优先注入他们，解决二义性问题。

    ​ 该方法适用于在某一个数据源是主数据源时

    /**
* foo数据源
*
* @return {@link DataSource }
* @author zhuzy
* @date 2022/07/17 20:01
*/
@Bean
@Primary
public DataSource fooDataSource() {
    DataSourceProperties dataSourceProperties = fooDataSourceProperties();
    log.info("foo dataSourceProperties: {}", dataSourceProperties.getUrl());
    return dataSourceProperties.initializeDataSourceBuilder().build();
}

  • 方法2：排除掉DataSourceAutoConfiguration、DataSourceTransactionManagerAutoConfiguration、JdbcTemplateAutoConfiguration，手动选择对应的数据源

// 多数据源需要排除数据源自动配置类,以及事务管理器自动配置类
@SpringBootApplication(exclude = {
        DataSourceAutoConfiguration.class,
        DataSourceTransactionManagerAutoConfiguration.class,
        JdbcTemplateAutoConfiguration.class
})
@Slf4j
public class Ge01DatasourceApplication implements CommandLineRunner {

    @Autowired
    @Qualifier("fooDataSource")
    private DataSource dataSource;
}

连接池

HikariCP

Spring Boot 2.0默认连接池，是一位在日本工作的白人大爷发明的，中文名为光，特点就是快

快的原因：

  • 很多api没有实现，基于字节码级别优化，通过JavaAssist生成
  • 大量小的改进
    • FastStatementList代替ArrayList
    • 无锁集合ConcurrentBag
    • 代理类优化，用invokestatic代替invokevitrual

Druid

特点：

  • 具备强大的监控功能
  • 针对主流数据库返回码都有支持
  • SQL防注入
  • 内置加密配置
  • 众多拓展点，方便进行定制

如何进行拓展：

​ 通过继承FilterEventAdapter，并修改META-INFO/druid-filter.properties

如何使用druid：

​ 需要在spring-boot-starter-jdbc中排除掉HikariCP，并导入依赖druid-srping-boot-starter

动态数据源

方式一：自定义DataSource

​ 通过自定义DataSource，对n个数据源进行聚合，并基于操作类型，返回对应的数据源。

@Repository
@Primary
public class DymicDataSource implements DataSource, InitializingBean {
    @Autowired
    @Qualifier("fooDataSource")
    private DataSource fooDataSource;

    @Autowired
    @Qualifier("barDataSource")
    private DataSource barDataSource;

    private ThreadLocal<String> operType = new ThreadLocal<>();

    @Override
    public Connection getConnection() throws SQLException {
        if ("foo".equals(operType.get())) {
            return fooDataSource.getConnection();
        } else {
            return barDataSource.getConnection();
        }
    }
}

缺点：DataSourc中有很多个api需要进行实现，直接实现DataSource而不实现其他方法完全是不行的，开发环境中也不可能采用这种方法

方式二：AbstractRoutingDataSource

​ 由于方式一的缺点显而易见，我们根本不需要关心很多其他的api，因此Spring对DataSource又进行了抽象，我们直接继承``即可实现上述功能

@Repository
@Primary
public class DymicDataSource02 extends AbstractRoutingDataSource {
    @Autowired
    @Qualifier("fooDataSource")
    private DataSource fooDataSource;

    @Autowired
    @Qualifier("barDataSource")
    private DataSource barDataSource;

    private ThreadLocal<String> operType = new ThreadLocal<>();

    @Override
    protected Object determineCurrentLookupKey() {
        return operType.get();
    }

    @Override
    public void afterPropertiesSet() {
        HashMap<Object, Object> targetDataSource = new HashMap<>();
        targetDataSource.put("foo", fooDataSource);
        targetDataSource.put("bar", barDataSource);

        // 初始化targetDataSource
        super.setTargetDataSources(targetDataSource);

        // 设置默认数据源
        super.setDefaultTargetDataSource(fooDataSource);

        super.afterPropertiesSet();
    }
}

​ 需要注意的是：

  • 数据源均存储在targetDataSource中，通过api - determineCurrentLookupKey，返回的标识码，返回targetDataSource对应的DataSource

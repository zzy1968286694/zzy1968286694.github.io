博客搭建01-使用GitHub + Hexo搭建博客

欢迎来到我的博客，本篇是博客的开篇，会介绍该博客是如何搭建起来的。

博客使用GitHub + Hexo进行搭建， 另外，使用到了hexo-theme-matery部署主题。网页是一个纯静态页面，因此，不需要服务器就能够发布部署以及访问，相当方便。搭建网站大概花了半天的时间，有兴趣的同学也可以自己尝试进行部署哦！

接下来让我详细介绍网站是如何一步步搭建部署的， 以及其中可能遇到的问题。

前置条件

  1. GitHub（GitHub注册及使用)
  2. Git (Git详细安装步骤)
  3. Node.js (Node.js详细安装步骤)

以上三个相信大家都有过接触，而且机器里有安装，如果没有的话，可以参考以上教程照着弄一下，由于比较简单，就不详细介绍了。

步骤1：创建Git仓库并配置Token

创建Git仓库

需要注意仓库名称的格式为Github账号用户名.github.io, Github仓库实际上是存储博客静态页面，以及博客的空间，由于有Github的存在，我们的博客才可以在不需要服务器的支持下，进行访问。

（这里由于我已经创建过了， 所以显示已存在）

配置Token

由于GitHub更换了认证方式，因此大家在提交代码的时候，需要使用Token进行提交（本质上hexo就是将GitHub作为远程仓库，存储网页和博客文档，因此也需要进行身份认证才能提交）

如何配置Token大家可以参考Github官方文档

提醒一下，生成后的Token记得先保存起来，因为生成后就不在可见了，另外，注意不要给Token太高的权限，否则别人拿到你的Token是可以随意修改你的仓库的！！！

步骤2：安装Hexo-cli并初始化博客

安装Hexo

和创建新项目一样，需要在本地磁盘新建一个文件夹Blog（文件夹名称随意），鼠标右键打开Git Bash Here，输入npm 命令安装 Hexo

npm install -g hexo-cli

初始化博客

  1. 使用命令hexo init进行初始化，是不是有点类似于git init

  1. 使用hexo g生成页面文件（生成网站静态文件到默认设置的 public 文件夹(hexo generate 的缩写)）

  2. 使用hexo s在本地运行（server）

  3. 此时，你已经可以在本地浏览你的博客网站啦！

​ 可以发现此时页面还比较丑，而且什么东西都没有，但是我们已经可以在本地进行访问了，接下来需要做的便是将其部署到GitHub，并配置好看的主题，让我们继续往下看吧！

步骤3：将博客部署到GitHub

​ 我们做博客网站最主要还是方便我们随时随地能够看到自己的笔记，并将其分享给其他同学，因此接下来要做的便是将博客网站部署到GitHub，这样所有人就都可以看到我们的博客啦！

​ 让我们回到Blog文件夹，用记事本打开_config.yml文件，如下图：

​ 我们只需要修改一个地方，即可让我们的博客配置到GitHub，下滑到文件的最下面，修改为以下内容

deploy:
  type: git
  repository: https://github.com/zzyong24/zzyong24.github.io.git  #你的仓库地址
  branch: main

这里需要注意一个点：repository地址的填写，建议使用ssh地址，这样就可以不用每次输入用户名和token了，直接可以配置ssh免密登陆

配置ssh免密步骤可参考Git密钥及仓库管理

(这里自己刚开始部署的时候也没用发现这个问题，后台每次hexo d都需要输入token，特别烦，所以查阅了一些资料

参考设置 SSH 使用 hexo deploy 时免输用户名密码

还需要注意一个问题，就是自己的分支需要看清楚是master还是main，（由于种种原因，master、slave等敏感词汇被禁用，你懂的）

接下来还是回到Blog文件夹，让我们发布到GitHub上吧~

打开Git Bash，安装Git部署插件，输入命令

npm install hexo-deployer-git --save

然后继续输入以下三条命令

hexo clean   #清除缓存文件 db.json 和已生成的静态文件 public
hexo g       #生成网站静态文件到默认设置的 public 文件夹
hexo d       #部署到设定的仓库(hexo deploy 的缩写)

完成以后，打开浏览器，输入 https://xxx.github.io就可以打开你的网页了, 比如我的博客地址为https://zzyong24.github.io/，如果你的博客没有显示或刷新可以等一小会儿再刷新一下，它可能会有延迟的问题。到这一步为止，你已经成功部署了属于你自己的一个简单博客了，不过可能不太好看，支持的功能也不够多，接下来我们使用主题hexo-theme-matery来美化一下我们的博客。

步骤4：使用hexo-theme-matery美化博客

前端工程化

1. Vue Cli初始化项目

  1. 安装vue cli

    npm install -g @vue/cli

  2. 创建新项目

    vue create hello-world

  3. 创建项目时，最好自定义创建，并选择vuex和vue-router

Yarn安装依赖

  1. 下载yarn(全局安装)

    npm install -g -y yarn

  2. 下载axios、qs

    ​ qs是是用于application/x-www-form-urlencoded格式传参，能够将对象转换为url参数（x=?&&y=?&&…），如果后台全部使用json传参，可以不需要安装

    yarn add axios qs

axios封装

  1. 基于webpack发布环境，设置base url

    通过process.env.NODE_ENV可以获取当前运行环境

    ​ 我们在npm运行或打包时，会自动设置当前环境，如：

    npm run dev：process.env.NODE_ENV为development

    npm run build：process.env.NODE_ENV为production

    // 基于webpack 发布环境，设置请求公共url
switch (process.env.NODE_ENV) {
    case "production":
        axios.defaults.baseURL="http://127.0.0.1:80";
        break;
    case "test":
        axios.defaults.baseURL="http://127.0.0.1:7777";
        break;
    default:
        axios.defaults.baseURL="http://127.0.0.1:8888";
}

    另外，可以在package.json中设置其他启动命令脚本，并设置启动脚本的环境，如下：

    "scripts": {
    "serve": "vue-cli-service serve",
    "serve:test": "set NODE_ENV=test&&vue-cli-service serve",
    "serve:production": "set NODE_ENV=production&&vue-cli-service serve",
    "build": "vue-cli-service build",
    "lint": "vue-cli-service lint"
  },

    同时，可以使用

    yarn serve/serve:test/serve:production

    等命令，启动服务

  2. 设置请求超时时间、并设置跨域允许携带凭证信息

    // 设置请求超时时间、跨域允许携带凭证等信息（cookie）
axios.defaults.timeout = 10000;
axios.defaults.withCredentials = true;

  3. 设置请求传递参数格式

    // 设置请求传递数据格式（配置情况1：后端接收参数格式为json）
axios.defaults.headers['Content-Type'] = 'application/json;charset=utf-8';
// 配置情况2：后端接收参数格式为表单形式 (x=?&&y=?&&...)
axios.defaults.headers['Content-Type'] = 'application/x-www-form-urlencoded';

  4. 设置post请求参数格式（和上面content-type需要配套）

    // 配置post请求参数的格式（配置情况1：后端接收参数格式为json）-- 无需配置
axios.defaults.transformRequest = data => {JSON.stringify(data)};
// 配置post请求参数的格式(配置情况2：后端接收参数格式为表单形式(x=?&&y=?&&...）
axios.defaults.transformRequest = data => {qs.stringify(data)};

  5. 设置请求拦截器

    /*
 设置请求拦截器
 作用：客户端发送请求 -> 请求拦截器 -> 服务器
 需要做的事情：
    1.token校验（JWT)： 接收服务器返回的token，存储在vuex/cookie/本地存储中，每次向服务器发送请求之前，都需要携带token信息
    2.重复请求过滤

 api参数：
    config：客户端发送请求所携带的config，包括参数、cookie、header等信息

 拦截请求配置项，并返回请求配置项
 */
axios.interceptors.request.use((config) => {
    // 判断是否需要设置token
    const isToken = (config.headers || {}).isToken === true;
    // 判断是否允许重复提交
    const canRepeatSubmit = (config.headers || {}).canRepeatSubmit === true;
    // 如果需要设置token，则获取token，并存储在headers中
    if (isToken) {
        // 在本地存储中获取token
        let token = localStorage.getItem('token');
        token && (config.headers.Authorization = token); // 设置token
    }

    // 判断请求方式，如果为get，需要将参数拼接在url后面
    if (config.method === 'get' && config.params) {
        let url = config.url + '?' + qs.stringify(config.params);
        config.url = url;
        config.params = {};
    }
    // 对于post请求、put请求，禁止重复提交
    if (!canRepeatSubmit && (config.method === 'post' || config.method === 'put')) {
        const requestParam = {
            url: config.url,
            data: typeof config.data === 'object' ? JSON.stringify(config.data) : config.data,
            time: new Date().getTime()
        }
        // 获取session中保存提交参数
        const sessionParam = cache.session.getJson('requestParam');
        if (sessionParam === undefined || sessionParam === null || sessionParam === '') {
            cache.session.setJson('requestParam', requestParam);
        } else {
            const s_url = sessionParam.url;
            const s_data = sessionParam.data;
            const s_time = sessionParam.time;
            const intervalTime = 1000;
            // 如果都相等，且间隔时间小于intervalTime，则判断为重复提交
            if (requestParam.data === s_data && requestParam.url === s_url && requestParam.time - s_time < intervalTime) {
                const message = '数据已提交，请勿重复提交';
                console.warn(`[${s_url}]:` + message)
                return Promise.reject(new Error(message));
            } else {
                cache.session.setJson('requestParam', requestParam);
            }
        }
    }
    return config;
}, error => {
    console.log(error);
    Promise.reject(error);
});

  6. 设置响应拦截器

    /*
 响应拦截器
 服务器响应信息 -> 响应拦截器 -> 客户端JS获取到数据
 */
axios.interceptors.response.use(response => {
    // config.data 即为后端返回值
    // 获取状态码，未返回默认为成功状态
    const code = response.data.code || 200;
    // 获取错误信息
    const msg = errorCode[code] || response.data.msg || errorCode["default"];
    // 判断数据格式，如果是二进制数据，直接返回
    if (response.request.responseType === 'blob' || response.request.responseType === 'arraybuffer') {
        return response.data;
    }
    if (code === 401) {
        MessageBox.confirm('登陆信息已过期，您可以继续留在该页面，或者重新登陆', '系统提示', {
                confirmButtonText: '重新登陆',
                cancelButtonText: '取消',
                type: 'warning'
            }
        ).then(() => {
            store.dispatch('LogOut').then(() => {
                location.href = '/index';
            })
        }).catch(() => {

        })
        return Promise.reject('无效的会话，或会话已过期，请重新登陆');
    } else if (code === 500) {
        Message({
            message: msg,
            type: 'error'
        })
        return Promise.reject(new Error(msg))
    } else if (code !== 200) {
        Notification.error({
            title: msg
        })
        return Promise.reject('error')
    } else {
        return response.data
    }
}, error => {
    console.log('err: ' + error)
    let {message} = error;
    if (message === "Network Error") {
        message = "后端接口连接异常";
    } else if (message.includes("timeout")) {
        message = "系统接口请求超时";
    } else if (message.includes("Request failed with status code")) {
        message = "系统接口" + message.substr(message.length - 3) + "异常";
    }
    Message({
        message: message,
        type: 'error',
        duration: 5 * 1000
    })
    return Promise.reject(error)
})

Vue项目架构

除去自动生成的，主要有以下几个目录：

  1. api：js函数文件
  2. components：组件
  3. plugins：插件函数js
  4. router：vue router配置
  5. store：vuex 配置
  6. utils：工具函数js
  7. vies: 界面

2. Vue配置

配置组件允许非驼峰命名

默认情况下，vue组件不能够是单字母的，并且得是驼峰命名。我们可以通过修改vue.config.js文件，修改其配置。

const { defineConfig } = require('@vue/cli-service')
module.exports = defineConfig({
  transpileDependencies: true,
  lintOnSave: false
})

4. VueX

VueX提供了一个容器store，它更方便的便于我们进行状态state的共享。

使用：

  1. 在vue cli初始化时，选择了vueX后，会有以下配置自动生成

    • ```javascript
      main.js

      import store from ‘./store’
      new Vue({
      router,
      store,
      render: h => h(App)
      }).$mount(‘#app’)

      
   - ![vuex](前端工程化/image-20220915221817073.png)

2. 在store中，我们可以配置state，即共享状态，并在index.js中将其暴露

   例如如下：配置了user state

   ```javascript
   import {getToken, setToken, removeToken} from "@/utils/auth";
   import { login, logout, getInfo } from '@/api/login'
   
   const user = {
       // vueX数据
       state: {
           token: getToken(),
           name: '',
           avatar: '',
           roles: [],
           permissions: []
       },
   
       // vueX修改数据
       mutations: {
           SET_TOKEN: (state, token) => {
               state.token = token;
           },
           SET_NAME: (state, name) => {
               state.name = name;
           },
           SET_AVATAR: (state, avatar) => {
               state.avatar = avatar;
           },
           SET_ROLES: (state, roles) => {
               state.roles = roles;
           },
           SET_PERMISSIONS: (state, permissions) => {
               state.permissions = permissions;
           }
       },
   
       // vueX异步修改数据
       actions: {
           // 登录
           Login({commit}, userInfo) {
               const username = userInfo.username.trim();
               const password = userInfo.password;
               const code = userInfo.code;
               const uuid = userInfo.uuid;
               return new Promise(((resolve, reject) => {
                   login(username, password, code, uuid).then(res => {
                       setToken(res.token);// 将token存储进cookie
                       commit('SET_TOKEN', res.token);// 调用maintainer方法，将token存储进vueX
                       resolve();
                   }).catch(error => {
                       reject(error);
                   })
               }))
           },
       }
   }
   
   export default user;

    ​ user中定义了多个state，mutations允许我们操作数据，actions允许我们异步操作数据。

5. Vue Router

Vue Router可以用于构建单页面应用，它很方便的能够帮助我们做页面的跳转

使用：

  1. 在vue cli初始化时，选择了Vue Router，会有以下初始化配置

    • ```javascript
      main.js

      import router from ‘./router’
      new Vue({
      router,
      store,
      render: h => h(App)
      }).$mount(‘#app’)

      
   - ![vue router](前端工程化/image-20220915233427013.png)

   

2. 定义router.index.js文件

   ```javascript
   import Vue from 'vue'
   import VueRouter from 'vue-router'
   import Login from '../views/login.vue'
   import HelloWorld from "@/components/HelloWorld";
   
   Vue.use(VueRouter)
   
   const routes = [
     {
       path: '/',
       name: 'login',
       component: Login
     },
     {
       path: '/helloWorld',
       name: 'hello',
       component: HelloWorld
     }
   ]
   
   const router = new VueRouter({
     routes
   })
   
   export default router

    ​ 我们定义了两个路由，分别为/以及/helloWorld,分别对应组件Login以及Helloworld跳转。

  2. 路由跳转

    • 标签跳转

      <router-link class="link-type" :to="'/register'">立即注册</router-link>

    • 代码跳转

      this.$router.push('/login')

CSS

background-size：设置背景图片的尺寸

length 设置背景图像的高度和宽度。第一个值设置宽度，第二个值设置高度。如果只设置一个值，则第二个值会被设置为 “auto”。
percentage 以父元素的百分比来设置背景图像的宽度和高度。第一个值设置宽度，第二个值设置高度。如果只设置一个值，则第二个值会被设置为 “auto”。
cover 把背景图像扩展至足够大，以使背景图像完全覆盖背景区域。背景图像的某些部分也许无法显示在背景定位区域中。
contain 把图像图像扩展至最大尺寸，以使其宽度和高度完全适应内容区域。

输入框跳转

表单输入中，可能需要监听enter便于输入框的跳转

使用时需要声明enter 事件，并给输入框注册上ref，跳转api传入参数ref

<el-input
    v-model="registerForm.username"
    type="text"
    auto-complete="off"
	placeholder="请输入用户名"
    ref="username"
    @keyup.enter.native="jumpNext('email')"
>
</el-input>

/**
* 跳转下一输入框
*/
jumpNext(jumpRef) {
    if (jumpRef === 'register') {
        this.register();
        return;
    }
    this.$refs[jumpRef].focus();
}

Jenkins

1. 所处地位与作用

​ Jenkins是一个开源的、提供友好操作界面的持续集成(CI)工具，起源于Hudson（Hudson是商用的），主要用于持续、自动的构建/测试软件项目、监控外部任务的运行。Jenkins用Java语言编写，可在Tomcat等流行的servlet容器中运行，也可独立运行。通常与版本管理工具(SCM)、构建工具结合使用。常用的版本控制工具有SVN、GIT，构建工具有Maven、Ant、Gradle。

3. 集成SVN

1. SVN安装及部署

1. docker安装

​ 为了方便演示，使用docker部署svn

  1. 拉取docker镜像

    docker pull docker.io/garethflowers/svn-server

  2. 启动svn容器

    docker run -v ~/docker-data/svn/repo:/var/opt/svn --name svn-server -p 3690:3690 --privileged=true -e SVN_REPONAME=repository -d docker.io/garethflowers/svn-server

    ​ 配置解释：

    ​ -v 磁盘挂载：将容器内文件/var/opt/svn文件夹下文件挂载到外部linux文件系统中（数据持久化）

    ​ -p端口映射：将容器内部端口对外映射到linux端口

    ​ --privileged：应用容器获取宿主机root权限

    ​ -e：设置容器内部环境变量

    ​ -d：指定镜像

  3. 进入容器内部进行配置

    docker exec -it svn-server /bin/sh

  4. 创建仓库

    svnadmin create /var/opt/svn/dasyRepo

  5. 进入仓库配置文件

    cd /var/opt/svn/dasyRepo/conf

  6. 编辑配置

    • svnserver.conf

      ​ authz-db = authz表示启用authz文件验证，开启这个配置后，authz的权限配置才有效

    • passwd

    • authz

  7. 退出重启svn-server

    docker restart svn-server

    ​ General

    配置项 含义
    描述 项目描述
    GitHub项目 是否为GitHub项目（配置项目url）
    This project is parameterized 项目参数化，可配置部署或启动参数
    Throttle builds 构建次数控制，可配置单位时间内最大构建次数
    丢弃旧的构建 配置构建记录、发布包保留的最大个数以及天数
    关闭构建 可以通过该配置关闭自动构建
    在必要时并发构建 允许同时构建该项目的多个版本（版本不一致、参数不一致）
    构建静默期 在触发构建后（SVN提交代码），允许等待静默期配置的时间再构建
    重试次数 检出代码失败后重试检出
    该项目的上游项目正在构建时阻止该项目构建 上游依赖项目构建时阻止构建
    该项目的下游项目正在构建时阻止该项目构建 下游被依赖项目构建时阻止构建
    使用自定义的工作空间 配置jenkins检出代码并构建代码的目录
    显示名称 显示该选项配置的名称
    保留构建的依赖日志 保存构建日志，不受其他删除配置的影响

    源码管理

    参数 含义
    Repository URL 仓库地址
    Credentials svn操作凭证
    Local module directory 检出目录位置
    Repository depth 检出深度（默认infinity无穷大）
    Check-out Strategy 检出策略

    触发远程构建 提供http接口方式远程构建项目的能力，需要配置token，请求时携带该token
    其他工程构建后触发 在其他工程构建后，该工程也开始构建（适用于该工程作为某工程的测试工程）
    定时构建 基于Crontab表达式定时构建项目（请思考是否需要启用该配置：CI目的便在于一发生修改便进行集成，若使用定时任务，可能和CI理念背道而驰）

5. 远程发布

​ 一般情况下，项目的构建地址可能位于jenkins，但是运行地址可能不一致，可能在测试服务器也可能在线上服务器。

​ 所以本地构造，远程发布是很常见的。

  1. 安装插件publish over ssh

  2. 重新打开项目配置，设置Post Steps配置send file or excute command over ssh

  3. 目前没有可选服务器，需要进行添加

    回到设置页，打开系统管理->系统设置，配置publish over ssh

​ 优化：

​ 执行命令时，可能会出现一些问题，例如由于导致之前的进程没关闭，端口被占用。所以我们可能需要写一些逻辑，执行一些其他的操作。

  1. 在pre steps，执行构建前删除kill jar（最好还是配在post steps里,和nohup写一起弄个脚本）

    #!/bin/bash
pid=`jps -l | grep 'jenkins*' | awk -F' ' '{print $1}'`
if [ -n "$pid" ];then
        echo "正在杀死pid：$pid" 
        kill -9 $pid
fi

  2. 在post steps，执行构建后，运行命令

    nohup java -jar jenkins-demo-0.0.1-SNAPSHOT.jar >mylog.log 2>&1 &

6.自动化构建

开启允许触发远程构建，并配置token（token不能带特殊符号）

url地址：JENKINS_URL/job/dasy-jenkins-demo/build?token=TOKEN_NAME

仅仅这样配置还不够，因为不安装插件的情况下，执行远程调用需要登录jenkins服务器

因此还需要安装插件Build Authorization Token Root允许使用token情况下，不登录触发远程部署

直接请求远程部署地址即可：

http://192.168.56.10:8080/buildByToken/build?job=NAME&token=TOKEN
